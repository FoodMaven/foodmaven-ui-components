<fm-characteristics-input> allows the user to select any number of characteristics as long as all the characteristics are compatible.

@Input() allCharacteristics:
{
   implicitCharacteristic: Characteristic[],
   scaledCharacteristic: Characteristic[],
   singleGroupCharacteristic: Characteristic[],
   multiGroupCharacteristic: Characteristic[]
}
This component also allows for the pre-population of characteristics. The identified characteristics must exist in the list of all characteristics in order to be automatically populated.

@Input() identifiedCharacteristics: IdentifiedCharacteristic[]

where an identified characteristic is in the shape of:

{
   externalCompId: string,
   name: string,
   value: string,
   type: string
}
There are no output events associated with this component. Form related functionality should be handled in the implementor. In order to access the currently selected characteristics, a ViewChild should be created for this component. The selectedOptions field should then be available to pull data from.

This component also allows for an optional readonly mode - where prepopulated characteristics cannot be removed and new characteristics cannot be added.
